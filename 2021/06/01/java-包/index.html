java-包

1. 包的导入

1.1. 类的导入

  • import java.time.LocalDate ;
  • 使用*引入包下的所有类： import java.util.*;
    • 只能使用星号导入一个包，而不能使用import java.* or import java.*.*导入以java为前缀的所有包
  • 在包中定位类是编译器(compiler )的工作。类文件中的字节码肯定使用完整的包名来引用其他类

1.2. 静态导入

import 语句不仅可以导入类,还增加了导入静态方法和静态域的功能。

2. 包的定义

2.1. 将类放入包中

  • package定义
  • 将源文件放到包名对应的子目录中

3. 包作用域

  • 没有被指定private or public的部分（类、方法、变量）可以被同一个包中的所有方法访问

4. class path

4.1. jar包

  • 在一个JAR文件中,可以包含多个压缩形式的类文件和子目录,这样既可以节省又可以改善性能
  • JAR文件使用ZIP格式组织文件和子目录

4.2. 虚拟机定位类

以class path是/home/user/classdir:.:/home/user/archives/archive.jar为例

  • 假定虚拟机要搜寻com.horstmann.corejava.Employee类文件（对于虚拟机来说类完整路径）：
    • 首先要查看存储在jre/lib和jre/lib/ext目录下的归档文件中所存放的系统类文件
      • 由于运行时库文件（rt.jar和在jre/lib与jre/lib/ext目录下的一些其他jar文件）会被自动地搜索,所以不必将它们显式地列在类路径中
    • /home/user/classdir/com.horstmann.corejava.Employee.class
    • 从当前目录开始：com.horstmann.corejava.Employee.class
    • com.horstmann.corejava.Employee.class inside/home/user/archives/archive.jar

4.3. 编译器定位类

即找到类所属的完整包名

  • 假设源文件包含：import java.util.* & import com.hortmann.corejava.*; 并且引用了Employee类

  • 编译器

    • 先试图查找java.lang.Employee（被默认导入）
    • java.util.Employee
    • com.hortmann.corejava.Empoyee
    • 和当前包中的Empolyee

  • 不唯一则编译出错

  • 如果import 的class的source file比current file要新（说明被import的内容有改动），则source file自动被重新编译

    • 可以看出有一个class -> source file寻找的过程：如果其他包public class，由于class需要与file文件名相同所以很容易找到；如果当前包非public class，则需要搜索当前包中的所有source file，来确定哪个source file定义了此类

4.4. 设置class path

  • java -classpath /home/user/classdir:.:/home/user/archives/archive.jar MyProg
  • export CLASSPATH=/home/user/classdir:.:/home/user/archives/archive.jar：直到shell退出有效

5. 补充编译相关

5.1. 单个源文件

  • 一个源文件只能有一个共有类，且文件名必须与public类名一致
  • 单文件有多个类时，javac EmpolyeeTest.java会生成两个类文件：EmployeeTest.class和Employee.class
    • 将程序中包含main方法的类名提供给字节码解释器，以便启动这个程序java Employeetest

5.2. 多源文件

许多程序员习惯于将每一个类存在一个单独的源文件中。例如，将 Employee类存放在文件 Employee.java中，将 EmployeeTest类存放在文件 EmployeeTest.java中。如果喜欢这样组织文件，将可以有两种编译源程序的方法。

  • 一种是使用通配符调用Java编译器：javac Employee*.java，于是所有与通配符匹配的源文件都将被编译成类文件。
  • 或者键入下列命令javac Employeetest.java
    • 并没有显式地编译 Employee.java。然而，当Java编译器发现 Employee Testjava使用了 Employee类时会查找名为 Employee.class的文件。如果没有找到这个文件，就会自动地搜索 Employee. java,然后，对它进行编译。更重要的是：如果 Employee.java版本较已有的 Employee.class文件版本新，Java编译器就会自动地重新编译这个文件。
