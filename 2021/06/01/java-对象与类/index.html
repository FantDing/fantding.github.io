java-对象与类

1. 概述

1.1. 类间关系

  • 依赖：uses-a

    • 这种使用关系从语义上说是，是具有偶然性的、临时性的、非常弱的，但是B类的变化会影响到A

    • 一般而言，依赖关系在Java中体现为局域变量、方法的形参，或者对静态方法的调用

  • 关联：

    • 这种关系比依赖更强、不存在依赖关系的偶然性、关系也不是临时性的，一般是长期性的，而且双方的关系一般是平等的、关联可以是单向、双向的。

    • 在Java中，关联关系一般使用成员变量来实现。

  • 聚合：has-a

    • 他体现的是整体与部分、拥有的关系，即has-a的关系。a是整体，b是部分。

    • 不同于关联关系的平等地位，聚合关系中两个类的地位是不平等。

  • 继承：is-a

1.2. 使用预定义类

1.2.1. 对象与对象变量

  • 关于Date birthday=new Date()的解释：
    • new操作符完成两件事：构造对象、返回引用（相当于C++中的对象指针）
    • birthday是对象变量，不是对象
      • 调用没有引用的对象变量，会产生编译错误
      • 调用null对象变量，会产生运行时错误

2. 用户自定义类

2.1. 成员变量（域）

2.1.1. 初始化方法

  • 构造器中初始化

    • 没有初始化，会默认赋值——不好的习惯：数值为0、布尔值为false，对象引用为null

  • 显式域初始化（声明时赋值）

    • private String nane ="";

    • 可以调用方法对域进行初始化

  • 初始化块。只要构造类的对象，这些块就会被执行。

初始化顺序

  1. 所有数据域被初始化为默认值(0、 false或null)
  2. 按照在类声明中出现的次序，依次执行所有域初始化语句和初始化块。
  3. 如果构造器第一行调用了第二个构造器，则执行第二个构造器主体。
  4. 执行这个构造器的主体

2.1.2. final

  • final表明对象变量不变，而引用的对象可能会改变
  • 声明为final的成员变量须在构造函数中初始化

2.3. 成员方法

  • 在Java中，所有的方法都必须在类的内部定义，但并不表示它们是内联方法

    • 是否将某个方法设置为内联方法是Java虚拟机的任务。即时编译器会监视调用那些简洁、经常被调用、没有被重载以及可优化的方法。

  • get & set方法

    • 注意：不要编写返回引用可变对象的访问器方法。在 Employee类中就违反了这个设计原则，其中的 gethireday方法返回了一个Date类对象

      • 外部调用Date类对象更改器方法，Date对象是可变的，这一点就破坏了封装性

      • 解决方法：使用clone

  • 参数：Java程序设计语言总是采用按值调用

    • 基本数据类型(数字、布尔值)
    • 对象引用——看作按值传递指针

2.4. 静态域与静态方法

2.4.1. 静态域

  • 静态成员变量：如nextId
  • 静态成员常量：用的较多，如public static final double PI =3.14159265358979323846
    • 然而，公有常量(即final域)却没问题。因为被声明为 final,所以，不允许外部改变

2.4.2. 静态方法

静态方法：C++中使用::操作符，Java中使用.操作符

  • 工厂方法

    • 使用构造器无法返回不同的类对象，而工厂方法可以

  • main方法

    • 每一个类可以有一个main方法。这是一个常用于对类进行单元测试的技巧。例可以在 Employee类中添加一个main方法。如果想要独立地测试 Employee类，只需要执行java employee；如果 Employee 类是一个更大型应用程序的一部分，就可以使用下面这条语句运行程序java Application，Employee类的main方法水远不会执行。
