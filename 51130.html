java-对象与类

1. 概述

1.1. 类间关系

  • 依赖：uses-a

    • 这种使用关系从语义上说是，是具有偶然性的、临时性的、非常弱的，但是B类的变化会影响到A

    • 一般而言，依赖关系在Java中体现为局域变量、方法的形参，或者对静态方法的调用

  • 关联：

    • 这种关系比依赖更强、不存在依赖关系的偶然性、关系也不是临时性的，一般是长期性的，而且双方的关系一般是平等的、关联可以是单向、双向的。

    • 在Java中，关联关系一般使用成员变量来实现。

  • 聚合：has-a

    • 他体现的是整体与部分、拥有的关系，即has-a的关系。a是整体，b是部分。

    • 不同于关联关系的平等地位，聚合关系中两个类的地位是不平等。

  • 继承：is-a

1.2. 使用预定义类

1.2.1. 对象与对象变量

  • 关于Date birthday=new Date()的解释：
    • new操作符完成两件事：构造对象、返回引用（相当于C++中的对象指针）
    • birthday是对象变量，不是对象
      • 调用没有引用的对象变量，会产生编译错误
      • 调用null对象变量，会产生运行时错误

2. 用户自定义类

2.1. 成员变量（域）

2.1.1. 初始化方法

  • 构造器中初始化

    • 没有初始化，会默认赋值——不好的习惯：数值为0、布尔值为false，对象引用为null

  • 显式域初始化（声明时赋值）

    • private String nane ="";

    • 可以调用方法对域进行初始化

  • 初始化块。只要构造类的对象，这些块就会被执行。

初始化顺序

  1. 所有数据域被初始化为默认值(0、 false或null)
  2. 按照在类声明中出现的次序，依次执行所有域初始化语句和初始化块。
  3. 如果构造器第一行调用了第二个构造器，则执行第二个构造器主体。
  4. 执行这个构造器的主体

  • 声明为final的成员变量须在构造函数中初始化

2.3. 成员方法

  • 在Java中，所有的方法都必须在类的内部定义，但并不表示它们是内联方法

    • 是否将某个方法设置为内联方法是Java虚拟机的任务。即时编译器会监视调用那些简洁、经常被调用、没有被重载以及可优化的方法。

  • get & set方法

    • 注意：不要编写返回引用可变对象的访问器方法。在 Employee类中就违反了这个设计原则，其中的 gethireday方法返回了一个Date类对象

      • 外部调用Date类对象更改器方法，Date对象是可变的，这一点就破坏了封装性

      • 解决方法：使用clone

  • 参数：Java程序设计语言总是采用按值调用

    • 基本数据类型(数字、布尔值)
    • 对象引用——看作按值传递指针

2.4. 静态域与静态方法

2.4.1. 静态域

  • 静态成员变量：如nextId

3. 继承

3.1. 子类定义

  • extends关键字

C++注释：Java与C++定义继承类的方式十分相似。Java用关键字 extends代替了C++中的冒号(：)。在Java中，所有的继承都是公有继承，而没有C++中的私有继承和保护继承

  • 继承层次：Java不支持多继承。Java中多继承功能由接口实现

3.2. 访问权限

  • 不能直接访问父类私有域。
    • 如子类getSalary()访问父类private salary：return salary+this.bonus;会出错；
    • 直接访问salary也会出错：return getSalary()+this.bonus;，因为getSalary()已经被重写了。
    • 要想访问父类域，需要使用super关键字：return super.getSalary()+this.bonus;

注释：有些人认为 super与this引用是类似的概念，实际上，这样比较并不太恰当。这是因为 super不是一个对象的引用，不能将 super赋给另一个对象变量，它只是一个指示编译器调用超类方法的特殊关键字。

3.3. 子类构造器

使用super调用父类构造器，实现对私有域的初始化

  • 使用super调用构造器的语句必须是子类构造器的第一条语句
  • 如果子类的构造器没有显式地调用超类的构造器，则将自动地调用超类默认（没有参数）的构造器。如果超类没有不带参数的构造器，并且在子类的构造器中又没有显式地调用超类的其他构造器，则Java编译器将报告错误。
  • 关键字this有两个用途：一是引用隐式参数，二是调用该类其他的构造器。
  • 同样， super关键字也有两个用途：一是调用超类的方法，二是调用超类的构造器。
  • 在调用构造器的时候，这两个关键字的使用方式很相似。调用构造器的语句只能作为另个构造器的第一条语句出现。构造参数既可以传通给本类(this)的其他构造器，也可以传递给超类( super)的构造器。
